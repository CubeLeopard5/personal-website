Rappel

Politique de confidentialité

Dernière mise à jour : 11 février 2026

Introduction

Rappel est une application de gestion de rappels personnels (factures, santé, courses, etc.) développée par Adrien Michaud. La protection de vos données personnelles est une priorité. Cette politique de confidentialité décrit quelles données sont collectées, comment elles sont stockées et utilisées.

En résumé : toutes vos données restent exclusivement sur votre appareil. Aucune donnée n'est transmise à des serveurs externes ni partagée avec des tiers.

Données collectées

L'application stocke les données suivantes localement sur votre appareil dans une base de données SQLite :

Profil utilisateur

Nom d'affichage, avatar, devise préférée, préférences de thème et paramètres de notification. Aucun mot de passe, adresse e-mail ou identifiant externe n'est collecté.

Rappels

Titre, date d'échéance, montant (optionnel), notes (optionnel), statut, récurrence, catégorie, priorité et étiquettes.

Organisation

Catégories, étiquettes et priorités personnalisées que vous créez pour organiser vos rappels.

Stockage des données

Stockage local uniquement

Toutes les données sont stockées exclusivement dans le stockage interne de votre appareil. L'application ne dispose d'aucun serveur distant, d'aucune base de données cloud et d'aucun système de synchronisation. Vos données ne quittent jamais votre appareil.

Isolation des profils

Si plusieurs profils sont créés sur le même appareil, les données de chaque profil sont strictement isolées les unes des autres.

Transmission des données

Aucune transmission

L'application ne transmet aucune donnée à des serveurs externes. Aucune API distante n'est appelée.

Aucun suivi

L'application n'intègre aucun outil d'analyse (analytics), de suivi comportemental ou de publicité.

Aucun partage

Vos données ne sont jamais partagées avec des tiers, des annonceurs ou des partenaires.

Permissions de l'application

L'application demande les permissions suivantes, toutes utilisées exclusivement pour des fonctionnalités locales :

Notifications (POST_NOTIFICATIONS)

Permet d'envoyer des notifications locales pour vous rappeler vos échéances. Les notifications sont gérées entièrement sur l'appareil, sans serveur de notifications distant.

Alarmes exactes (SCHEDULE_EXACT_ALARM)

Permet de planifier des notifications précises à l'heure exacte de vos rappels.

Démarrage (RECEIVE_BOOT_COMPLETED)

Permet de reprogrammer vos notifications après un redémarrage de l'appareil.

Internet et réseau (INTERNET, ACCESS_NETWORK_STATE)

Requis par le framework .NET MAUI. L'application ne transmet aucune donnée via le réseau.

Notifications

Les notifications sont générées et affichées localement sur votre appareil. Par mesure de sécurité :

Protection des données sensibles

Les montants financiers et les détails médicaux ne sont jamais affichés dans les notifications. Seul le titre du rappel apparaît dans la notification.

Sauvegardes

Sur Android, la sauvegarde automatique de l'appareil peut inclure les données de l'application si vous avez activé la sauvegarde cloud dans les paramètres de votre appareil. Cette sauvegarde est gérée par le système d'exploitation Android et Google, et non par l'application Rappel.

Suppression des données

Vous avez un contrôle total sur vos données :

Suppression individuelle

Vous pouvez supprimer à tout moment un rappel, une catégorie, une étiquette ou une priorité depuis l'application.

Suppression complète

La suppression d'un profil entraîne la suppression définitive de toutes les données associées. La désinstallation de l'application supprime également toutes les données locales.

Enfants

L'application Rappel n'est pas destinée aux enfants de moins de 13 ans et ne collecte pas sciemment de données les concernant. L'application ne nécessite aucune création de compte et ne collecte aucune information personnelle identifiable.

Modifications de cette politique

Cette politique de confidentialité peut être mise à jour. Toute modification sera publiée sur cette page avec une date de mise à jour actualisée. Nous vous encourageons à consulter régulièrement cette page.
